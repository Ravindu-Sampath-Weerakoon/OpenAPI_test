ESP32 Vehicle Submodule API Documentation

This documentation is for a vehicle submodule developed using one ESP32 configured as a Wi-Fi Access Point. The ESP32 hosts a web server on port 80 and returns JSON data for the main controller board.

Wi-Fi Access Point Server: http://192.168.4.1 Port: 80 Response Type: JSON

Integration Steps

  1. Power on the ESP32 vehicle submodule.
  2. Connect the main controller board or test device to the ESP32 Access Point.
  3. Send an HTTP GET request to http://192.168.4.1/.
  4. Read the returned JSON response.
  5. Use status, data.canOperate, data.warnings, and error to decide the controller action.

Warning vs Error Rule

Type Meaning Controller Action
Success Module works normally. Continue operation.
Warning Module can still work, but there is a condition to monitor. Continue operation and display/log warning.
Error Module cannot safely work. Stop or ignore this module until fixed.
