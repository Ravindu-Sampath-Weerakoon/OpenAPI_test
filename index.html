Vehicle Body Control API Documentation

Complete technical reference for the ESP32 Vehicle Body Control JSON API.

Wi-Fi Station Mode JSON Response Port 80

Integration Steps

  1. Power on the ESP32 module.
  2. Ensure it is connected to the Wi-Fi network (SSID: Dialog 4G 048).
  3. Obtain the ESP32 IP Address from the Serial Monitor.
  4. Send a GET request to http://<ESP32_IP>/.

Full JSON Example

{
  "module": "Door, Centre Lock, Power Mirror and Rear Boot Door Control",
  "status": "success",
  "message": "Vehicle body control data retrieved successfully",
  "driveStatus": "SAFE_TO_DRIVE",
  "riskMessage": "No body control risk detected.",
  "data": {
    "front_left_door_status": "closed",
    "front_right_door_status": "closed",
    "rear_left_door_status": "closed",
    "rear_right_door_status": "closed",
    "centre_lock_status": "locked",
    "power_mirror_left_position": 12,
    "power_mirror_right_position": -5,
    "rear_boot_door_status": "closed",
    "error_status": "OK"
  },
  "error": {
    "code": 0,
    "message": "None"
  }
}

Variable Dictionary

  • driveStatusString
    Critical safety indicator. If SAFE_TO_DRIVE, the vehicle can proceed. If RISK_DETECTED, a safety issue exists.
  • power_mirror_*_positionInteger
    Mirror offset from center. Positive = Up/Right, Negative = Down/Left, 0 = Center.
  • error_statusString
    Internal sensor health. "OK" means all hardware sensors are functioning.
