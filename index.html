Vehicle Body Control API Documentation

Detailed field-by-field breakdown of the ESP32 Vehicle Body Control JSON response.

Wi-Fi Station Mode JSON API Port 80

Variable Dictionary (Root Level)

  • moduleString
    Identifies the hardware module sending the data. Used for routing in multi-module systems.
    Value: "Door, Centre Lock, Power Mirror and Rear Boot Door Control"
  • statusEnum
    Indicates if the API call was successful.
    Possible Values: "success", "error"
  • driveStatusString
    The most critical field for the main vehicle controller. Summarizes safety.
    Values: "SAFE_TO_DRIVE", "RISK_DETECTED"
  • riskMessageString
    A detailed explanation of why the vehicle might be unsafe to drive.

Variable Dictionary (Data Object)

  • front_left_door_statusString
    Physical sensor state of the driver-side front door.
    Values: "open", "closed"
  • centre_lock_statusString
    State of the electronic central locking mechanism.
    Values: "locked", "unlocked"
  • power_mirror_left_positionInteger
    The angular/linear offset of the mirror.
    Logic: (+) Up/Right, (-) Down/Left, (0) Centered.
  • error_statusString
    Internal sensor health status reported by the ESP32.
    Example: "OK", "SENSOR_FAULT"

(Note: All other door/mirror fields follow the same logic as the examples above.)

JSON Example

{
  "module": "Door, Centre Lock, Power Mirror and Rear Boot Door Control",
  "status": "success",
  "driveStatus": "SAFE_TO_DRIVE",
  "data": {
    "front_left_door_status": "closed",
    "centre_lock_status": "locked",
    "power_mirror_left_position": 12,
    "error_status": "OK"
  },
  "error": { "code": 0, "message": "None" }
}
